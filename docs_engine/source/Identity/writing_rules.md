# Writing Rules

> Part of **Product Identity** — see `identity_overview.md`.  
> Tone: `voice.md`. Terms: `vocabulary.md`. Quotes: `product_quotes.md`.

Rules for all future user-facing text in MapSnap.

Applies to:

- UI labels and messages
- Onboarding
- Dialogs and permission prompts
- Empty states
- Error and success feedback
- Website copy
- README
- App store description
- Documentation aimed at users

---

## Purpose

MapSnap copy is sparse and purposeful. Every word earns its place. Copy supports the experience — it does not explain the product.

---

## Prefer

- **Short sentences** — one breath, one idea
- **Active voice** — "Tryck SNAP" not "SNAP kan tryckas"
- **One idea at a time** — never stack instructions
- **Official vocabulary** — see `vocabulary.md`
- **Calm tone** — see `voice.md`

---

## Avoid

- Feature explanations — the interface should be self-evident
- Marketing language — no selling the app
- Long paragraphs — break or cut
- Multiple exclamation marks — one is already too many
- Hype words — "fantastic", "amazing", "revolutionary"
- Generic map-app language — see `vocabulary.md`

---

## When Possible

**Show, don't explain.**

A button label, icon, or empty state should communicate intent without a tutorial.

If text is needed, say the minimum. Remove words until meaning breaks, then add one back.

---

## Quotes

When using quotes from `product_quotes.md`:

- One quote per screen, normally
- Quotes support — they do not interrupt
- When in doubt, remove the quote

---

## Language

- User-facing copy: **Swedish**
- Identity documents: **English** (internal governance)

Swedish copy must still follow voice, vocabulary, and these rules.

---
