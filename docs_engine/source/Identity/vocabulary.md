# Vocabulary

> Part of **Product Identity** — see `identity_overview.md`.

Official MapSnap terminology. Vocabulary is part of product identity — not implementation detail.

User-facing copy is in Swedish. English terms below are the canonical product language; Swedish equivalents follow where established.

---

## Purpose

Consistent words create a coherent product. MapSnap has its own language. Use it everywhere — UI, documentation, store listings, and support.

---

## Official Terms

| Term | Meaning | Swedish (UI) |
|------|---------|--------------|
| **Snap** | The core action — one tap to capture a place | SNAP |
| **Snappar** | Saved captures; the user's collection | snappar |
| **Discovery** | A place worth remembering — not a generic pin | upptäckt |
| **Moment** | The experience at capture time — what happened there | ögonblick |
| **Capture** | The act of saving coordinates (via Snap) | fånga |
| **Return** | Going back to a saved place | återvända |
| **Snaptiser** | A reminder or intention attached to a Snap (provisional term) | snaptiser |
| **CAPTURE** | Pillar — save the place immediately | fånga |
| **REMEMBER** | Pillar — add meaning after capture | kom ihåg |
| **DELIGHT** | Pillar — restrained polish without slowing capture | — |

Use **Snap** as a verb and noun. It is the product's central word.

**Snaptiser** status: provisional working term — see `snaptiser_doctrine.md`. Requires explicit product approval before store/marketing use.

**Snappar** is the plural for saved items in user-facing Swedish copy.

**Discovery** frames what the user saves — something found, not something filed.

**Moment** refers to the lived experience, not metadata.

**Capture** describes the action; **Return** describes revisiting.

---

## Never Use

Unless technically necessary (code, APIs, third-party integration), do not use:

| Avoid | Use instead |
|-------|-------------|
| Bookmark | Snap |
| POI | Discovery / plats |
| Record | Snap / Capture |
| Object | Snap / Discovery |
| Save location | Snap / Fånga platsen |

Generic map-app or database language dilutes MapSnap's identity.

---

## Code vs Copy

Internal code may use technical names (`SnapPlace`, `snaps` store). User-facing text must follow this vocabulary.

When a technical term appears in error messages or permissions, prefer the closest official term and keep language plain.

---

## Adding Terms

New vocabulary requires explicit product approval. Terms must be timeless, minimal, and aligned with `product_doctrine.md` and `voice.md`.
